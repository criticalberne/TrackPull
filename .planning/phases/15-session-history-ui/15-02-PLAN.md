         - If false: keep existing `EXPORT_CSV_REQUEST` behavior unchanged

If the deleted session is the one currently being viewed (`viewingHistoricalSession` and `cachedData?.report_id === reportId`), call `dismissHistoricalSession()` first.
             - If target has id `clear-all-history-btn`: call `event.stopPropagation()`, show `confirm("Delete all saved sessions? This cannot be undone.")`, if confirmed call `clearAllHistory()` then `renderHistoryList()`. If `viewingHistoricalSession`, call `dismissHistoricalSession()` first.

        b. **Banner dismiss** on `#history-banner-dismiss`: call `dismissHistoricalSession()`

        c. **Initial history list render**: call `renderHistoryList()`

        - AFTER existing updates: call `renderHistoryList()` to refresh the history list (new capture may have added an entry)

    - Use `event.stopPropagation()` on delete button to prevent row click from firing
    - Use event delegation on `#history-list-content` container (not per-row handlers) since rows are dynamically rendered

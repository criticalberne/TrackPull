  <name>Task 2: Create options page TypeScript with CRUD logic and AI preference persistence</name>
  <files>src/options/options.ts</files>
  <action>
    Replace the placeholder `src/options/options.ts` with the full options page logic. Follow the DOMContentLoaded async IIFE pattern established in popup.ts.

    **Imports:**
    ```typescript
    import { BUILTIN_PROMPTS } from "../shared/prompt_types";
    import type { CustomPrompt } from "../shared/prompt_types";
    import { loadCustomPrompts, saveCustomPrompt, deleteCustomPrompt } from "../shared/custom_prompts";
    import { STORAGE_KEYS } from "../shared/constants";
    ```

    **Initialization (DOMContentLoaded):**
    ```typescript
    document.addEventListener("DOMContentLoaded", async () => {
      renderBuiltInPrompts();
      await renderCustomPrompts();
      setupNewPromptForm();
      await restoreAiPreference();
    });
    ```

    **renderBuiltInPrompts():**
    - Get `#builtin-prompts-list` container
    - For each prompt in BUILTIN_PROMPTS, create a read-only item showing:
      - Prompt name (bold)
      - Tier badge (small label: "Beginner", "Intermediate", or "Advanced")
      - Optionally show a collapsed preview of the template (Claude's discretion on display style -- collapsed list, cards, or simple rows are all acceptable)
    - Items are NOT editable, no action buttons

    **renderCustomPrompts():**
    - Get `#custom-prompts-list` container
    - Clear container innerHTML
    - Call `loadCustomPrompts()` to get saved custom prompts
    - For each custom prompt, create an item showing:
      - Prompt name
      - "Edit" button — clicking it shows the inline form pre-filled with the prompt's name and template, hides the "+ New Prompt" button, and sets a module-level `editingPromptId` variable
      - "Delete" button — calls `window.confirm("Delete this prompt?")` per user decision; if confirmed, calls `deleteCustomPrompt(id)` then re-renders
    - If no custom prompts exist, show a subtle "No custom prompts yet" message

    **setupNewPromptForm():**
    - Get the `#new-prompt-btn`, `#prompt-form`, `#save-prompt-btn`, `#cancel-prompt-btn`, `#prompt-name-input`, `#prompt-template-input`
    - `#new-prompt-btn` click: show form, hide button, clear inputs, set `editingPromptId = null`
    - `#cancel-prompt-btn` click: hide form, show button, clear inputs, set `editingPromptId = null`
    - `#save-prompt-btn` click:
      1. Read name and template from inputs
      2. Validate: name must not be empty, template must not be empty. Show toast error if invalid.
      3. If `editingPromptId` is not null, reuse the existing ID. Otherwise, generate new ID via `crypto.randomUUID()`.
      4. Create `CustomPrompt` object: `{ id, name: nameValue.trim(), template: templateValue }`
      5. Call `saveCustomPrompt(prompt)` wrapped in try/catch
      6. On success: show success toast, hide form, show button, re-render custom prompts
      7. On error: check if message includes "QUOTA_BYTES" — show "Storage full. Delete prompts to save new ones." toast; otherwise show generic error toast

    **restoreAiPreference():**
    - Get `#options-ai-service` select element
    - Read from chrome.storage.sync: `STORAGE_KEYS.AI_SERVICE`
    - If saved value exists, set select.value
    - Add change listener: on change, save to chrome.storage.sync

    **showToast(message, type):**
    - Replicate the exact showToast pattern from popup.ts (or extract shared logic — but simpler to duplicate for options page since they're separate bundles)
    - Get `#toast-container`, create div with class `toast ${type}`, append, auto-remove after 3s (success) or 5s (error)

    **Module-level state:**
    ```typescript
    let editingPromptId: string | null = null;
    ```

    **Edit flow detail:**
    When "Edit" is clicked on a custom prompt row:
    1. Set `editingPromptId = prompt.id`
    2. Fill `#prompt-name-input` with `prompt.name`
    3. Fill `#prompt-template-input` with `prompt.template`
    4. Show `#prompt-form`, hide `#new-prompt-btn`
    5. Save button now updates the existing prompt (same ID) rather than creating a new one

    **Anti-patterns to avoid:**
    - Do NOT use `options_page` (MV2 field) — the manifest already has `options_ui`
    - Do NOT store built-in prompts to storage — they are read from BUILTIN_PROMPTS constant only
    - Do NOT hardcode prompt data — iterate over BUILTIN_PROMPTS array

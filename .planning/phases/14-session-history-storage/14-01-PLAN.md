<feature>
  <name>Session History Storage Module</name>
  <files>src/models/types.ts, src/shared/constants.ts, src/shared/history.ts, tests/test_history.ts</files>
  <behavior>
    - saveSessionToHistory(session) strips raw_api_data and stores as SessionSnapshot wrapped in HistoryEntry with captured_at timestamp
    - saveSessionToHistory(session) with existing report_id: replaces the entry in place and refreshes captured_at to current time
    - saveSessionToHistory(session) with new report_id when 20 entries exist: evicts the oldest (by captured_at) and adds the new one
    - saveSessionToHistory(session) with new report_id when fewer than 20 entries: appends without eviction
    - getHistoryErrorMessage(error) returns "Storage full" message when error contains "QUOTA" or "quota"
    - getHistoryErrorMessage(error) returns generic fallback for unknown errors
    - SessionSnapshot type is Omit<SessionData, 'raw_api_data'> — raw_api_data field must not exist on snapshots
    - History array is sorted newest-first (descending captured_at) before writing
  </behavior>
  <implementation>
    1. Add types to src/models/types.ts (after SessionData):
       - `export type SessionSnapshot = Omit<SessionData, 'raw_api_data'>;`
       - `export interface HistoryEntry { captured_at: number; snapshot: SessionSnapshot; }`

    2. Add storage key to src/shared/constants.ts:
       - Add `SESSION_HISTORY: "sessionHistory"` to STORAGE_KEYS object

    3. Create src/shared/history.ts with:
       - `const MAX_SESSIONS = 20;`
       - `function createSnapshot(session: SessionData): SessionSnapshot` — destructure to exclude raw_api_data
       - `export function saveSessionToHistory(session: SessionData): Promise<void>` — reads SESSION_HISTORY from chrome.storage.local, deduplicates by report_id (full replace), sorts by captured_at descending, slices to MAX_SESSIONS, writes back. Rejects on chrome.runtime.lastError.
       - `export function getHistoryErrorMessage(error: string): string` — maps error substrings to user-friendly text. Check for "QUOTA_BYTES" or "quota" -> "Storage full -- oldest sessions will be cleared". Fallback: "Could not save to session history".

    4. Create tests/test_history.ts with chrome.storage.local mock:
       - Mock chrome.storage.local.get and chrome.storage.local.set as vi.fn()
       - Mock chrome.runtime.lastError as null by default
       - Test: saves new session (empty history -> 1 entry with snapshot lacking raw_api_data)
       - Test: deduplicates by report_id (2 saves with same report_id -> 1 entry, captured_at updated)
       - Test: evicts oldest when at 20-session cap (21st save removes oldest captured_at)
       - Test: re-captured report_id gets newest captured_at (won't be evicted soon)
       - Test: getHistoryErrorMessage maps quota errors correctly
       - Test: getHistoryErrorMessage returns fallback for unknown errors
       - Test: rejects promise on chrome.runtime.lastError

    Note on chrome mock: Define a global `chrome` object in the test file with `vi.stubGlobal('chrome', { storage: { local: { get: vi.fn(), set: vi.fn() } }, runtime: { lastError: null } })`. The get mock should call its callback with the stored data; the set mock should call its callback after capturing written data.
  </implementation>
</feature>

# Codebase Structure

## Directory Purposes

**src/background:**
- Purpose: Service worker for Chrome extension; central message hub and storage manager
- Contains: `serviceWorker.ts` only
- Key files: `src/background/serviceWorker.ts` — Handles SAVE_DATA, GET_DATA, EXPORT_CSV_REQUEST messages; manages chrome.storage.local; downloads CSV

**src/content:**
- Purpose: Content scripts injected into trackmangolf.com; data extraction and message routing
- Contains: Three scripts for different purposes
- Key files:
  - `src/content/interceptor.ts` — Monkey-patches fetch/XMLHttpRequest, parses API responses, scrapes DOM tags
  - `src/content/bridge.ts` — Forwards data from MAIN world to service worker via chrome.runtime.sendMessage
  - `src/content/html_scraping.ts` — Fallback DOM parser for when API unavailable

**src/models:**
- Purpose: TypeScript interfaces and data structures
- Contains: `types.ts` only
- Key files: `src/models/types.ts` — Shot, ClubGroup, SessionData interfaces; mergeSessionData() utility

**src/popup:**
- Purpose: Extension popup UI
- Contains: TypeScript logic and HTML template
- Key files:
  - `src/popup/popup.ts` — DOM event handlers, storage queries, UI updates
  - `src/popup/popup.html` — HTML template (bundled with popup.js at build time)

**src/shared:**
- Purpose: Reusable utility modules
- Contains: CSV writing, unit conversion, HTML parsing, constants
- Key files:
  - `src/shared/csv_writer.ts` — SessionData → CSV conversion with unit normalization
  - `src/shared/unit_normalization.ts` — Distance/angle/speed conversions; metric classification
  - `src/shared/html_table_parser.ts` — TableParser class for DOM extraction
  - `src/shared/constants.ts` — Metric lists, CSS selectors, storage key names

**src/icons:**
- Purpose: Extension icon assets
- Contains: PNG image files
- Key files: `src/icons/icon16.png` (referenced in manifest)

**dist/:**
- Purpose: Built extension ready for loading into Chrome
- Contains: Compiled .js bundles, manifest.json, popup.html, icons
- Generated by: `bash scripts/build-extension.sh` (esbuild)
- Files:
  - `dist/background.js` — Bundled service worker
  - `dist/interceptor.js` — Bundled MAIN world content script
  - `dist/bridge.js` — Bundled ISOLATED world content script
  - `dist/html_scraping.js` — Bundled fallback scraper
  - `dist/popup.js` — Bundled popup logic
  - `dist/popup.html` — Popup template
  - `dist/manifest.json` — Extension definition
  - `dist/icons/` — Icon assets

**tests/:**
- Purpose: Vitest unit test suite
- Contains: Test files matching `test_*.ts` pattern (gitignored; regenerated from src/)
- Test files:
  - `tests/test_csv_writer_units.ts` — CSV export with unit conversions
  - `tests/test_html_table_parser.ts` — TableParser DOM extraction
  - `tests/test_unit_normalization.ts` — Distance/angle/speed conversions
  - `tests/test_multi_page_merge.ts` — mergeSessionData() multi-page scenarios
  - `tests/test_extract_club_and_metrics.ts` — HTML table extraction
  - `tests/test_toast_error_handling.ts` — Toast message behavior
  - `tests/test_storage_keys.ts` — Storage constant names

**scripts/:**
- Purpose: Build automation
- Key files:
  - `scripts/build-extension.sh` — Main build script; esbuild bundles, copies assets, validates HTML
  - `scripts/build-zip.sh` — Creates production.zip for release
  - `scripts/smoke-test-extension.sh` — Load dist/ into Chrome for manual testing

## Key File Locations

**Entry Points:**

- `src/background/serviceWorker.ts` — Service worker main; chrome.runtime.onInstalled + message listener IIFE
- `src/content/interceptor.ts` — MAIN world self-executing IIFE; patches fetch + XHR
- `src/content/bridge.ts` — ISOLATED world window.addEventListener for postMessage
- `src/popup/popup.html` → `src/popup/popup.ts` — Popup UI DOMContentLoaded entry

**Configuration:**

- `src/manifest.json` — Chrome extension manifest (v3); defines permissions, content scripts, background service worker, popup action
- `vitest.config.ts` — Test runner config (include pattern: `tests/test_*.ts`)
- `package.json` — Project metadata, devDependencies (typescript, vitest)

**Core Logic:**

- `src/models/types.ts` — SessionData aggregate, ClubGroup, Shot definitions
- `src/content/interceptor.ts` — API response parsing, shot data extraction
- `src/shared/csv_writer.ts` — SessionData serialization
- `src/shared/unit_normalization.ts` — Unit conversions and metric classification

**Testing:**

- `tests/test_*.ts` — Vitest suites (test naming convention required by vitest.config.ts)
- Test file location mirrors src/ structure conceptually (not co-located)

## Naming Conventions

**Files:**

- `camelCase.ts` for TypeScript sources (e.g., `serviceWorker.ts`, `csv_writer.ts`)
- `UPPERCASE.md` for planning documents (e.g., `ARCHITECTURE.md`)
- `test_*.ts` for test files (not `*.test.ts`) — required by vitest.config.ts

**Directories:**

- kebab-case for grouped modules: `src/content/`, `src/shared/`, `src/popup/`
- Single-purpose directories (background, models, popup) are singular

**Classes:**

- PascalCase: TableParser, TableRow, TableCell, TableData, TableExtractionResult

**Functions:**

- camelCase: parseSessionData(), handleCapturedJson(), scrapeGroupTags(), convertDistance()

**Constants:**

- UPPER_SNAKE_CASE: METRIC_KEYS, CSS_RESULTS_WRAPPER, STORAGE_KEYS, DEFAULT_UNIT_CHOICE

**Interfaces/Types:**

- PascalCase: SessionData, ClubGroup, Shot, UnitChoice, UnitSystem

## Where to Add New Code

**New Feature (e.g., new metric type):**

1. Update metric allowlist in `src/content/interceptor.ts` METRIC_KEYS set
2. Add to `src/shared/constants.ts` ALL_METRICS, METRIC_DISPLAY_NAMES, METRIC_GROUPS if needed
3. Update `src/shared/unit_normalization.ts` if metric needs unit conversion (DISTANCE_METRICS, ANGLE_METRICS, SPEED_METRICS)
4. Update CSV column order in `src/shared/csv_writer.ts` METRIC_COLUMN_ORDER if desired
5. Write test in `tests/test_*.ts` (follow existing patterns)

**New Content Script (e.g., intercept new API endpoint):**

- Add .ts file to `src/content/` (e.g., `src/content/my_interceptor.ts`)
- Add to manifest.json content_scripts array
- Add build step to `scripts/build-extension.sh` using esbuild

**New Shared Utility Module (e.g., data validation):**

- Create `src/shared/my_utility.ts`
- Export public functions, import in dependent modules (e.g., interceptor.ts, popup.ts)
- Add unit tests to `tests/test_my_utility.ts`

**New UI Component (popup extension):**

- Add HTML to `src/popup/popup.html`
- Add event handler functions to `src/popup/popup.ts`
- Import needed utilities (e.g., STORAGE_KEYS from constants.ts)
- Test by rebuilding with `bash scripts/build-extension.sh` and checking popup.html validity

## Special Directories

**dist/:**
- Purpose: Built artifact directory (tracked in git; must rebuild after src/ changes)
- Generated: Yes (by esbuild in build-extension.sh)
- Committed: Yes (must commit after build for releases)
- Contents: .js bundles, manifest.json, popup.html, icons — ready to load into Chrome

**tests/:**
- Purpose: Vitest test files
- Generated: No (manually written)
- Committed: No (gitignored; test files exist locally for development)
- Running: `npx vitest run` (uses vitest.config.ts include pattern)

**build-output/:**
- Purpose: Intermediate build artifacts (icons preprocessing, etc.)
- Generated: Yes (during build)
- Committed: No (gitignored)

- Purpose: Standalone Python tools (not part of Chrome extension)
- Not deployed with extension
- Separate from src/

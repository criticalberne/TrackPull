Edit `src/manifest.json` to make two changes:

**Change 1: Version bump**
Change `"version": "1.4.0"` to `"version": "1.5.0"`.

**Change 2: Add commands section**
Add a `"commands"` key to the top-level object. Place it AFTER `"host_permissions"` and BEFORE `"background"` for logical grouping (permissions-related keys together). The commands section is:

```json
"commands": {
  "_execute_action": {
    "suggested_key": {
      "default": "Ctrl+Shift+G",
      "mac": "Command+Shift+G"
    },
    "description": "Open TrackPull"
  }
},
```

The `_execute_action` reserved command name tells Chrome to open the extension popup when the shortcut is pressed. No background.js handler is needed -- Chrome handles it natively.

The `suggested_key` object uses platform-specific keys:
- `"default": "Ctrl+Shift+G"` covers Windows and Linux
- `"mac": "Command+Shift+G"` covers macOS

The `description` field appears in `chrome://extensions/shortcuts` to help users identify the shortcut.

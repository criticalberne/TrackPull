# Plan 05-03: Prompt Builder Module

## Goal

Create a `prompt_builder.ts` module with a pure function `assemblePrompt()` that combines a prompt template with TSV data and optional session metadata into a final clipboard-ready string. This is the last piece that connects the prompt library (Plan B) with the TSV writer (Plan A) for Phase 6 popup integration.

## Tasks

<task id="C1" title="Create src/shared/prompt_builder.ts with assemblePrompt function">
Create the prompt builder module at `src/shared/prompt_builder.ts`.

Imports:
- `import type { BuiltInPrompt } from "./prompt_types"`

Define types:
- `export interface PromptMetadata` with fields:
  - `date: string` -- session date (e.g., "2025-01-15")
  - `shotCount: number` -- total number of shots in the session
  - `unitLabel: string` -- human-readable unit description (e.g., "mph + yards", "m/s + meters")

Export functions:

1. `export function assemblePrompt(prompt: BuiltInPrompt, tsvData: string, metadata?: PromptMetadata): string`
   - If `metadata` is provided, build a context header line: `Session: {date} | {shotCount} shots | Units: {unitLabel}`
   - Build the data block: if metadata exists, it is `contextHeader + "\n\n" + tsvData`; if no metadata, it is just `tsvData`
   - Replace `{{DATA}}` in `prompt.template` with the data block
   - Return the assembled string

2. `export function buildUnitLabel(unitChoice: { speed: string; distance: string }): string`
   - Returns a human-readable string like `"mph + yards"` or `"m/s + meters"`
   - Used by popup.ts (Phase 6) to construct the `unitLabel` field for `PromptMetadata`

3. `export function countSessionShots(session: { club_groups: Array<{ shots: unknown[] }> }): number`
   - Sums `shots.length` across all `club_groups`
   - Used by popup.ts (Phase 6) to construct the `shotCount` field for `PromptMetadata`
   - Uses a minimal type signature to avoid importing the full `SessionData` type

All functions are pure (no side effects, no storage access). Named exports only.
</task>

<task id="C2" title="Create tests/test_prompt_builder.ts with unit tests" depends_on="C1">
Create the test file at `tests/test_prompt_builder.ts` using vitest `describe/it/expect`.

Import `assemblePrompt`, `buildUnitLabel`, `countSessionShots`, `PromptMetadata` from `../src/shared/prompt_builder`.
Import `BuiltInPrompt` from `../src/shared/prompt_types`.

Create a `makePrompt` helper that returns a minimal `BuiltInPrompt` object with a configurable template string.

Test groups:

**describe("assemblePrompt: basic assembly")**
- `it("replaces {{DATA}} with TSV data")` -- template `"Here is data:\n\n{{DATA}}"`, verify TSV appears in output
- `it("returns template unchanged when {{DATA}} is missing")` -- template without placeholder, verify original template returned
- `it("handles empty TSV data string")` -- verify `{{DATA}}` is replaced with empty string

**describe("assemblePrompt: with metadata")**
- `it("prepends metadata header before TSV data")` -- pass metadata, verify output contains `Session: 2025-01-15 | 10 shots | Units: mph + yards`
- `it("separates metadata header from TSV with blank line")` -- verify `\n\n` between metadata line and first TSV line
- `it("omits metadata header when metadata is undefined")` -- no metadata arg, verify no "Session:" line in output

**describe("assemblePrompt: integration with real prompts")**
- `it("works with a BUILTIN_PROMPTS entry")` -- import `BUILTIN_PROMPTS` from prompt_types, assemble the first prompt with sample TSV, verify output contains both prompt text and data
- `it("assembled output contains no {{DATA}} placeholder")` -- verify the placeholder is fully replaced

**describe("buildUnitLabel")**
- `it("returns 'mph + yards' for imperial")` -- `buildUnitLabel({ speed: "mph", distance: "yards" })`
- `it("returns 'm/s + meters' for metric")` -- `buildUnitLabel({ speed: "m/s", distance: "meters" })`
- `it("returns 'mph + meters' for hybrid")` -- `buildUnitLabel({ speed: "mph", distance: "meters" })`

**describe("countSessionShots")**
- `it("counts total shots across all clubs")` -- 2 clubs with 3 and 2 shots -> 5
- `it("returns 0 for empty session")` -- empty `club_groups` array -> 0
</task>

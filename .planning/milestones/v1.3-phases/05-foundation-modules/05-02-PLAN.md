# Plan 05-02: Prompt Library Types and Constants

## Goal

Create the `prompt_types.ts` module containing the `BuiltInPrompt` type, the `SkillTier` type, and a `BUILTIN_PROMPTS` constant array with 7+ golf analysis prompts organized by skill tier. Prompts are TypeScript constants only -- they do not go into chrome.storage.

## Tasks

<task id="B1" title="Create src/shared/prompt_types.ts with types and 7+ built-in prompts">
Create the prompt types and library module at `src/shared/prompt_types.ts`.

Define types:
- `export type SkillTier = "beginner" | "intermediate" | "advanced"`
- `export interface BuiltInPrompt` with fields:
  - `readonly id: string` -- kebab-case identifier (e.g., `"session-overview-beginner"`)
  - `readonly name: string` -- human-readable display name (e.g., `"Session Overview"`)
  - `readonly tier: SkillTier`
  - `readonly topic: string` -- category identifier (e.g., `"overview"`, `"club-breakdown"`)
  - `readonly template: string` -- prompt text containing `{{DATA}}` placeholder

Define `export const BUILTIN_PROMPTS: readonly BuiltInPrompt[]` with at least 8 prompts.

Required prompts (from user decisions in CONTEXT.md):

1. **Session Overview (Beginner)** -- `id: "session-overview-beginner"`, topic: `"overview"`
   Friendly coach tone. "Tell me how I did today." Ask for 2-3 positives and 1-2 things to work on. Encourage the player. Include `{{DATA}}` placeholder.

2. **Club-by-Club Breakdown (Intermediate)** -- `id: "club-breakdown-intermediate"`, topic: `"club-breakdown"`
   Compare performance across clubs. Identify strengths, weaknesses, and distance gaps. Moderate technical depth.

3. **Consistency Analysis (Advanced)** -- `id: "consistency-analysis-advanced"`, topic: `"consistency"`
   Data-driven analyst tone. Analyze shot dispersion, standard deviations, and repeatability patterns. Numbers-first approach.

Additional prompts (Claude's discretion, reaching 7+ total):

4. **Launch & Spin Optimization (Intermediate)** -- `id: "launch-spin-intermediate"`, topic: `"launch-spin"`
   Analyze launch angle, spin rate, and spin axis combinations. Compare to optimal windows for each club type. Suggest adjustments.

5. **Distance Gapping Report (Beginner)** -- `id: "distance-gapping-beginner"`, topic: `"distance-gapping"`
   Friendly tone. Look at carry and total distances per club. Identify gaps between clubs. Suggest whether club selection is working.

6. **Shot Shape & Dispersion (Intermediate)** -- `id: "shot-shape-intermediate"`, topic: `"shot-shape"`
   Analyze side, curve, face angle, and club path. Characterize the player's typical shot shape. Identify patterns and misses.

7. **Club Delivery Analysis (Advanced)** -- `id: "club-delivery-advanced"`, topic: `"club-delivery"`
   Technical analyst tone. Analyze attack angle, club path, face angle, face-to-path, dynamic loft. Numbers-first, correlate delivery metrics to ball flight outcomes.

8. **Quick Session Summary (Beginner)** -- `id: "quick-summary-beginner"`, topic: `"quick-summary"`
   Very short, friendly summary. Just the headlines: best club, longest shot, most consistent club. 3-4 bullet points max.

Prompt writing guidelines:
- Beginner: friendly coach tone ("Nice work on X", "Let's sharpen Y"), simple language, limit technical terms
- Intermediate: moderate depth, explain what metrics mean, actionable suggestions
- Advanced: technical analyst tone, data-driven, numbers-first, reference specific metric names
- All prompts end with a data section: the template includes `{{DATA}}` where the TSV data will be injected
- Each prompt should instruct the AI to reference the tabular data provided
- Prompts should mention that the data is tab-separated Trackman golf session data

Use `as const` for the array to enable narrowing.

No default exports. No barrel files.
</task>

<task id="B2" title="Create tests/test_prompt_types.ts with validation tests" depends_on="B1">
Create the test file at `tests/test_prompt_types.ts` using vitest `describe/it/expect`.

Import `BUILTIN_PROMPTS`, `BuiltInPrompt`, `SkillTier` from `../src/shared/prompt_types`.

Test groups:

**describe("Prompt Library: structure")**
- `it("has at least 7 built-in prompts")` -- `BUILTIN_PROMPTS.length >= 7`
- `it("every prompt has a unique id")` -- collect all ids into a Set, verify Set size equals array length
- `it("every prompt has a non-empty name")` -- all names are truthy strings
- `it("every prompt has a valid tier")` -- all tiers are one of `"beginner"`, `"intermediate"`, `"advanced"`
- `it("every prompt has a non-empty topic")` -- all topics are truthy strings
- `it("every prompt template contains {{DATA}} placeholder")` -- all templates include `"{{DATA}}"`

**describe("Prompt Library: tier distribution")**
- `it("includes at least one beginner prompt")` -- filter by tier, verify length >= 1
- `it("includes at least one intermediate prompt")` -- filter by tier, verify length >= 1
- `it("includes at least one advanced prompt")` -- filter by tier, verify length >= 1

**describe("Prompt Library: required topics")**
- `it("includes a session overview prompt")` -- at least one prompt with topic "overview"
- `it("includes a club breakdown prompt")` -- at least one prompt with topic "club-breakdown"
- `it("includes a consistency analysis prompt")` -- at least one prompt with topic "consistency"

**describe("Prompt Library: template quality")**
- `it("every template is at least 50 characters")` -- minimum meaningful prompt length
- `it("no template exceeds 2000 characters")` -- reasonable upper bound to keep clipboard payload manageable
